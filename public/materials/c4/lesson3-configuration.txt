======================================

שיעור 3: Configuration Options
=================================

מה הן Configuration Options?
=============================

Configuration Options הן פרמטרים שמאפשרים לך לשלוט בהתנהגות המודל:
• יצירתיות (Temperature)
• מגוון (Top-P, Top-K)
• אורך תשובה (Max Output Tokens)
• עצירה (Stop Sequences)

למה זה חשוב?
=============

• שליטה על התנהגות המודל
• התאמה למשימה ספציפית
• אופטימיזציה של ביצועים
• חיסכון בעלויות

1. Temperature
===============

מה זה?
-------
Temperature שולטת על רמת היצירתיות והאקראיות של המודל.

טווח: 0.0 - 2.0
• 0.0 = דטרמיניסטי (תמיד אותה תשובה)
• 1.0 = מאוזן
• 2.0 = מאוד יצירתי (אקראי)

מתי להשתמש?
------------

Temperature 0.0 - 0.3:
• משימות עובדתיות
• כתיבת קוד
• תרגום
• סיכום
• שאלות ותשובות

דוגמה:
```python
config = {
    "temperature": 0.1,
    "model": "gemini-1.5-flash"
}

# מתאים לכתיבת קוד מדויק
```

Temperature 0.7 - 1.0:
• כתיבה יצירתית
• יצירת תוכן
• סיפורים
• רעיונות

דוגמה:
```python
config = {
    "temperature": 0.8,
    "model": "gemini-1.5-pro"
}

# מתאים לכתיבה יצירתית
```

Temperature 1.5 - 2.0:
• סיעור מוחות
• רעיונות חדשניים
• ניסויים
• יצירתיות מקסימלית

דוגמה:
```python
config = {
    "temperature": 1.8,
    "model": "gemini-1.5-pro"
}

# מתאים לרעיונות חדשניים
```

Best Practices:
---------------
• התחל עם 0.7 (ברירת מחדל)
• הורד ל-0.1-0.3 למשימות עובדתיות
• העלה ל-1.0+ ליצירתיות
• בדוק עם ערכים שונים

2. Top-P (Nucleus Sampling)
============================

מה זה?
-------
Top-P מגבילה את מגוון הטוקנים לפי הסתברות מצטברת.

טווח: 0.0 - 1.0
• 0.1 = מגוון קטן (חזוי)
• 0.5 = מאוזן
• 0.95 = מגוון רחב

איך זה עובד?
-------------
• בוחרים טוקנים עד שהסתברות מצטברת מגיעה ל-Top-P
• Top-P 0.9 = בוחרים טוקנים עד 90% הסתברות

מתי להשתמש?
------------

Top-P 0.1 - 0.3:
• משימות דטרמיניסטיות
• תשובות מדויקות
• קוד

דוגמה:
```python
config = {
    "top_p": 0.2,
    "temperature": 0.1
}

# תשובות חזויות ומדויקות
```

Top-P 0.5 - 0.7:
• משימות מאוזנות
• רוב השימושים

דוגמה:
```python
config = {
    "top_p": 0.6,
    "temperature": 0.7
}

# מאוזן בין יצירתיות לדיוק
```

Top-P 0.9 - 1.0:
• יצירתיות מקסימלית
• מגוון רחב

דוגמה:
```python
config = {
    "top_p": 0.95,
    "temperature": 1.5
}

# מגוון רחב של תשובות
```

קשר ל-Temperature:
------------------
• Temperature גבוה + Top-P גבוה = יצירתי מאוד
• Temperature נמוך + Top-P נמוך = דטרמיניסטי

Best Practices:
---------------
• השתמש ב-Top-P יחד עם Temperature
• Top-P 0.95 הוא ברירת מחדל טובה
• הורד ל-0.5-0.7 לדיוק
• העלה ל-0.9+ ליצירתיות

3. Top-K
=========

מה זה?
-------
Top-K מגבילה את מספר הטוקנים שנבחרים לפי הסתברות.

טווח: 1 - 40
• 1 = הכי סביר בלבד
• 10 = 10 הטובים ביותר
• 40 = מגוון רחב

מתי להשתמש?
------------

Top-K 1 - 5:
• תשובות חזויות
• דיוק מקסימלי

דוגמה:
```python
config = {
    "top_k": 3,
    "temperature": 0.1
}

# תשובות מאוד חזויות
```

Top-K 10 - 20:
• מאוזן
• רוב השימושים

דוגמה:
```python
config = {
    "top_k": 15,
    "temperature": 0.7
}

# מאוזן
```

Top-K 30 - 40:
• מגוון רחב
• יצירתיות

דוגמה:
```python
config = {
    "top_k": 40,
    "temperature": 1.2
}

# מגוון רחב
```

קשר ל-Top-P:
------------
• Top-K ו-Top-P עובדים יחד
• Top-K מגביל מספר, Top-P מגביל הסתברות
• בדרך כלל משתמשים באחד מהם

Best Practices:
---------------
• Top-K 40 הוא ברירת מחדל טוב
• הורד ל-10-20 לדיוק
• העלה ל-30-40 ליצירתיות
• בדרך כלל עדיף Top-P על Top-K

4. Max Output Tokens
=====================

מה זה?
-------
Max Output Tokens מגביל את אורך התשובה המקסימלי.

טווח: 1 - 8192 (תלוי במודל)
• 100 = תשובות קצרות מאוד
• 1000 = פסקה
• 8192 = מסמך ארוך

מתי להשתמש?
------------

Max Output Tokens 100 - 500:
• תשובות קצרות
• סיכומים
• תגובות מהירות

דוגמה:
```python
config = {
    "max_output_tokens": 200,
    "temperature": 0.3
}

# תשובות קצרות וממוקדות
```

Max Output Tokens 1000 - 2000:
• תשובות מפורטות
• הסברים
• רוב השימושים

דוגמה:
```python
config = {
    "max_output_tokens": 1500,
    "temperature": 0.7
}

# תשובות מפורטות
```

Max Output Tokens 4000 - 8192:
• מסמכים ארוכים
• תוכן מפורט מאוד
• דוחות

דוגמה:
```python
config = {
    "max_output_tokens": 6000,
    "temperature": 0.8
}

# מסמכים ארוכים
```

Best Practices:
---------------
• הגדר לפי הצורך האמיתי
• אל תגדיר גבוה מדי (חיסכון בעלויות)
• בדוק מה אורך אידיאלי
• השתמש ב-Stop Sequences לעצירה מוקדמת

5. Stop Sequences
=================

מה זה?
-------
Stop Sequences הם רצפי תווים שגורמים למודל להפסיק ליצור.

מתי להשתמש?
------------
• כשאתה יודע מתי התשובה מסתיימת
• למניעת תשובות ארוכות מדי
• לשליטה על פורמט

דוגמאות:
---------

עצירה בסוף פסקה:
```python
config = {
    "stop_sequences": ["\n\n---\n\n"],
    "max_output_tokens": 2000
}

# עוצר אחרי פסקה
```

עצירה אחרי רשימה:
```python
config = {
    "stop_sequences": ["\n\n## סיכום"],
    "max_output_tokens": 1500
}

# עוצר לפני סיכום
```

עצירה אחרי מספר שורות:
```python
config = {
    "stop_sequences": ["\n\n---\n\n", "\n\n##"],
    "max_output_tokens": 1000
}

# עוצר אחרי מספר סימנים
```

Best Practices:
---------------
• השתמש ב-Stop Sequences לשליטה על פורמט
• בדוק שהן לא חותכות תשובות חשובות
• הוסף כמה אפשרויות
• בדוק עם דוגמאות

דוגמאות קוד
============

Python:
-------
```python
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY")

model = genai.GenerativeModel(
    model_name="gemini-1.5-flash",
    generation_config={
        "temperature": 0.7,
        "top_p": 0.95,
        "top_k": 40,
        "max_output_tokens": 2000,
        "stop_sequences": ["\n\n---"]
    }
)

response = model.generate_content("כתוב סיכום על AI")
print(response.text)
```

JavaScript:
------------
```javascript
const { GoogleGenerativeAI } = require("@google/generative-ai");

const genAI = new GoogleGenerativeAI("YOUR_API_KEY");
const model = genAI.getGenerativeModel({
    model: "gemini-1.5-flash",
    generationConfig: {
        temperature: 0.7,
        topP: 0.95,
        topK: 40,
        maxOutputTokens: 2000,
        stopSequences: ["\n\n---"]
    }
});

const result = await model.generateContent("כתוב סיכום על AI");
console.log(result.response.text());
```

cURL:
-----
```bash
curl -X POST \
  'https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "contents": [{
      "parts": [{"text": "כתוב סיכום על AI"}]
    }],
    "generationConfig": {
      "temperature": 0.7,
      "topP": 0.95,
      "topK": 40,
      "maxOutputTokens": 2000,
      "stopSequences": ["\n\n---"]
    }
  }'
```
